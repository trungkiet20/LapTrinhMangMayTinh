JavaScript: Promise và Async/Await

October 17, 2025

Trong JavaScript, Promise và async/await là hai công cụ mạnh mẽ để xử lý bất đồng bộ — tức là những tác vụ mất thời gian như tải dữ liệu, truy vấn API, hoặc đọc file.

1. Vấn đề của callback

Trước khi có Promise, JavaScript dùng callback để xử lý bất đồng bộ. Nhưng callback lồng nhau dễ gây ra “callback hell”.

getUser(id, function(user) {
  getPosts(user.id, function(posts) {
    getComments(posts[0].id, function(comments) {
      console.log(comments);
    });
  });
});

Đoạn trên khó đọc và khó bảo trì. Promise ra đời để khắc phục điều này.

2. Promise là gì?

Promise là một đối tượng đại diện cho kết quả tương lai của một tác vụ bất đồng bộ. Nó có 3 trạng thái:

  • pending – đang chờ xử lý.
  • fulfilled – đã thành công.
  • rejected – đã thất bại.
const myPromise = new Promise((resolve, reject) => {
  setTimeout(() => {
    const success = true;
    if (success) resolve("Thành công!");
    else reject("Lỗi rồi!");
  }, 1000);
});

myPromise
  .then(result => console.log(result))
  .catch(error => console.error(error));

Ở ví dụ trên, Promise sẽ in ra “Thành công!” sau 1 giây.

3. Promise chaining (chuỗi then)

Bạn có thể nối nhiều Promise để xử lý tuần tự:

getData()
  .then(data => processData(data))
  .then(result => saveResult(result))
  .catch(error => console.error(error));

Ưu điểm: tránh lồng nhiều hàm và dễ đọc hơn.

4. Async/Await – Cú pháp hiện đại

async/await giúp viết Promise giống như mã đồng bộ, dễ hiểu hơn.

function fakeFetch() {
  return new Promise(resolve => {
    setTimeout(() => resolve("Dữ liệu tải xong!"), 2000);
  });
}

async function getData() {
  console.log("Đang tải...");
  const result = await fakeFetch();
  console.log(result);
}

getData();

await tạm dừng hàm cho đến khi Promise hoàn tất, sau đó trả về kết quả. Lưu ý: chỉ dùng await bên trong hàm có từ khóa async.

5. Xử lý lỗi với try...catch

async function loadData() {
  try {
    const res = await fakeFetch();
    console.log("Kết quả:", res);
  } catch (err) {
    console.error("Lỗi:", err);
  }
}

Dùng try...catch giúp bạn dễ dàng bắt lỗi mà không cần viết .catch() dài dòng.

6. Ví dụ thực tế: Giả lập Fetch API

function fakeAPI(url) {
  return new Promise((resolve, reject) => {
    console.log("Đang gọi API:", url);
    setTimeout(() => {
      if (url === "/users") resolve(["Kiệt", "Lan", "Nam"]);
      else reject("Không tìm thấy dữ liệu!");
    }, 1500);
  });
}

async function main() {
  try {
    const users = await fakeAPI("/users");
    console.log("Người dùng:", users);
  } catch (e) {
    console.error("Lỗi:", e);
  }
}

main();

7. Tổng kết

  • Promise giúp thay thế callback lồng nhau.
  • then() / catch() giúp quản lý chuỗi bất đồng bộ.
  • async/await giúp code gọn và dễ đọc hơn.

Trong bài sau, chúng ta sẽ tìm hiểu Fetch API và cách sử dụng Promise để kết nối tới dữ liệu thật.
