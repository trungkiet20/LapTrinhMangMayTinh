Java: Lập trình hướng đối tượng - Lớp và đối tượng

October 17, 2025

Lập trình hướng đối tượng (OOP - Object Oriented Programming) là nền tảng quan trọng trong Java. Trong bài này, chúng ta sẽ tìm hiểu về hai khái niệm cơ bản nhất: Lớp (Class) và Đối tượng (Object).

1. Khái niệm Lớp (Class)

Lớp là bản thiết kế (blueprint) mô tả thuộc tính và hành vi của một nhóm đối tượng. Nó giống như khuôn mẫu để tạo ra các đối tượng cụ thể.


public class Person {
    // Thuộc tính (fields)
    String name;
    int age;

    // Phương thức (methods)
    void sayHello() {
        System.out.println("Xin chào, tôi tên là " + name);
    }
}

Trong ví dụ trên, Person là một lớp có hai thuộc tính (name, age) và một phương thức sayHello().

2. Khái niệm Đối tượng (Object)

Đối tượng là một thể hiện (instance) của lớp. Khi bạn tạo đối tượng từ lớp, Java cấp phát bộ nhớ và cho phép bạn truy cập các thuộc tính, phương thức của nó.


public class Main {
    public static void main(String[] args) {
        // Tạo đối tượng từ lớp Person
        Person p1 = new Person();
        p1.name = "Kiệt";
        p1.age = 21;

        // Gọi phương thức
        p1.sayHello();
    }
}

Kết quả khi chạy chương trình:


Xin chào, tôi tên là Kiệt

3. Thuộc tính và Phương thức

Thuộc tính mô tả dữ liệu của đối tượng (như tên, tuổi). Phương thức định nghĩa hành vi (như nói, chạy, hiển thị thông tin).

Bạn có thể thêm phương thức khởi tạo (constructor) để gán giá trị ban đầu cho đối tượng:


public class Person {
    String name;
    int age;

    // Constructor
    public Person(String n, int a) {
        name = n;
        age = a;
    }

    void display() {
        System.out.println(name + " (" + age + " tuổi)");
    }
}

public class Main {
    public static void main(String[] args) {
        Person p = new Person("Minh", 22);
        p.display();
    }
}

Kết quả:


Minh (22 tuổi)

4. Tổng kết

  • Lớp (Class) là khuôn mẫu để tạo đối tượng.
  • Đối tượng (Object) là thể hiện cụ thể của lớp.
  • Thuộc tính biểu diễn dữ liệu, Phương thức biểu diễn hành vi.
  • Java sử dụng OOP để giúp mã nguồn dễ mở rộng, bảo trì và tái sử dụng.

Trong bài tiếp theo, chúng ta sẽ học về Kế thừa (Inheritance) và Đa hình (Polymorphism) — hai khái niệm nâng cao giúp OOP trở nên mạnh mẽ hơn.
