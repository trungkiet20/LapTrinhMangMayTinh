Java: Xử lý ngoại lệ và Logging

October 17, 2025

Trong quá trình lập trình, việc xảy ra lỗi (error) là điều không thể tránh khỏi. Java cung cấp cơ chế Exception Handling giúp lập trình viên kiểm soát luồng thực thi khi có lỗi và phản hồi hợp lý. Ngoài ra, ta có thể ghi lại thông tin lỗi bằng Logging để tiện theo dõi và gỡ lỗi sau này.

1. Cấu trúc try-catch-finally

Khối try chứa đoạn mã có thể phát sinh lỗi, catch dùng để xử lý lỗi, còn finally luôn được thực thi dù có lỗi hay không.


public class DivideExample {
    public static void main(String[] args) {
        try {
            int a = 10;
            int b = 0;
            int result = a / b;
            System.out.println("Kết quả: " + result);
        } catch (ArithmeticException e) {
            System.out.println("Lỗi: Không thể chia cho 0!");
        } finally {
            System.out.println("Khối finally luôn chạy.");
        }
    }
}

Khi chạy, chương trình sẽ bắt lỗi chia cho 0 và không bị dừng đột ngột.

2. Tạo ngoại lệ tùy chỉnh

Trong nhiều trường hợp, ta muốn tạo loại lỗi riêng phù hợp với nghiệp vụ của mình. Có thể tạo một lớp kế thừa từ Exception:


class InvalidAgeException extends Exception {
    public InvalidAgeException(String message) {
        super(message);
    }
}

public class CustomExceptionExample {
    static void checkAge(int age) throws InvalidAgeException {
        if (age < 18)
            throw new InvalidAgeException("Tuổi phải lớn hơn hoặc bằng 18!");
    }

    public static void main(String[] args) {
        try {
            checkAge(15);
        } catch (InvalidAgeException e) {
            System.out.println("Lỗi: " + e.getMessage());
        }
    }
}

Khi chạy, chương trình sẽ in ra lỗi do tuổi nhỏ hơn 18.

3. Logging trong Java

Logging giúp bạn ghi lại thông tin chạy chương trình (thông báo, cảnh báo, lỗi, v.v.). Java có sẵn gói java.util.logging hoặc có thể dùng thư viện phổ biến như SLF4J kết hợp với Logback.


import java.util.logging.Logger;

public class LoggingExample {
    private static final Logger logger = Logger.getLogger(LoggingExample.class.getName());

    public static void main(String[] args) {
        logger.info("Ứng dụng bắt đầu chạy...");
        try {
            int result = 10 / 0;
        } catch (Exception e) {
            logger.severe("Lỗi: " + e.getMessage());
        }
        logger.info("Ứng dụng kết thúc.");
    }
}

Đầu ra sẽ hiển thị thông tin log trên console hoặc file (nếu cấu hình).

4. Kết luận

Cơ chế Exception Handling giúp chương trình Java ổn định hơn bằng cách tránh crash đột ngột. Việc bổ sung Logging giúp theo dõi, gỡ lỗi và bảo trì dễ dàng hơn trong môi trường thực tế.
