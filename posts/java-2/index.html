Java: Phân biệt lớp và đối tượng (Class và Object)

Tác giả: Lại Ngọc Trung Kiệt

October 20, 2025

Java là một ngôn ngữ hướng đối tượng, và để thực sự làm chủ nó bạn cần hiểu rõ khái niệm “lớp” (class) và “đối tượng” (object) — hai thành phần cơ bản nhưng rất quan trọng.
Bài viết này sẽ giúp bạn phân biệt rõ ràng giữa lớp và đối tượng, cách tạo và sử dụng chúng trong Java, thông qua các ví dụ thực tế và minh hoạ chi tiết.

1. Khái niệm và cách tạo lớp & đối tượng (Class và Object)

1.1 Lớp (Class) là gì?

Trong Java, một lớp là một “khuôn mẫu” hoặc bản thiết kế cho các đối tượng. Nó định nghĩa các thuộc tính (fields) và hành vi (methods) mà các đối tượng cùng loại sẽ có.
Cú pháp khai báo lớp:

class ClassName {
    /* fields */
    /* methods */
}

Trong đó:

  • class: từ khóa để khai báo lớp.
  • ClassName: tên lớp – theo quy tắc PascalCase (chữ đầu mỗi từ viết hoa).
  • fields: biến thành viên của lớp.
  • methods: phương thức hoạt động thuộc lớp.

1.2 Đối tượng (Object) là gì?

Một đối tượng là một thể hiện cụ thể của một lớp — được tạo ra từ lớp bằng từ khóa new, có bộ nhớ riêng, mang các giá trị cụ thể cho các thuộc tính và có thể thực thi các phương thức của lớp.

Cách tạo đối tượng:

ClassName referenceVariable = new ClassName();

Trong đó:

  • ClassName: tên lớp.
  • referenceVariable: biến tham chiếu tới đối tượng.
  • new: từ khóa để khởi tạo đối tượng.
  • ClassName(): gọi đến constructor mặc định của lớp.

2. Sự khác biệt giữa lớp và đối tượng (Class vs Object)

STT Lớp (Class) Đối tượng (Object)
1 Là bản thiết kế hoặc khuôn mẫu từ đó đối tượng được tạo Là thể hiện cụ thể của lớp
2 Là tập hợp các đối tượng có thuộc tính và hành vi giống nhau Là một thực thể riêng biệt trong thế giới thực hoặc phần mềm
3 Khai báo một lần Có thể tạo nhiều đối tượng từ cùng một lớp
4 Được khai báo với từ khóa class Được khởi tạo với từ khóa new
5 Không chiếm bộ nhớ khi chỉ khai báo Chiếm bộ nhớ khi được tạo
6 Là thực thể logic Là thực thể vật lý (về mặt lập trình)

3. Ví dụ cụ thể về lớp và đối tượng (Class và Object)

Giả sử bạn xây dựng hệ thống quản lý “Chó” trong phần mềm. Bạn sẽ cần lưu thông tin như giống, tuổi, kích cỡ, màu sắc…
⇒ Hãy mô hình hoá bằng Java:

// Khai báo lớp
public class Dog {
    // Các biến thành viên
    String breed;
    String size;
    int age;
    String color;

    // Phương thức lấy thông tin của chó
    public String getInfo() {
        return "Breed is: " + breed + " Size is: " + size + " Age is: " + age + " Color is: " + color;
    }
}

Tạo đối tượng từ lớp Dog và gán giá trị:

public class Main {
    public static void main(String[] args) {
        Dog maltese = new Dog();
        maltese.breed = "Maltese";
        maltese.size = "Small";
        maltese.age = 2;
        maltese.color = "White";

        System.out.println(maltese.getInfo());
    }
}

Kết quả khi chạy chương trình sẽ là:

Breed is: Maltese Size is: Small Age is: 2 Color is: White

4. Ví dụ: Tách lớp khai báo & lớp thực thi

Trong ví dụ trên, chúng ta có lớp Dog chứa định nghĩa và phương thức, và lớp Main chịu trách nhiệm thực thi. Đây là cách tốt để tách rõ ràng trách nhiệm giữa mô hình và thực thi:

// Class khai báo
class Dog {
    String breed;
    String size;
    int age;
    String color;

    public String getInfo() {
        return "Breed is: " + breed + " Size is: " + size + " Age is: " + age + " Color is: " + color;
    }
}

// Class thực thi
public class Execute {
    public static void main(String[] args) {
        Dog maltese = new Dog();
        maltese.breed = "Maltese";
        maltese.size = "Small";
        maltese.age = 2;
        maltese.color = "White";
        System.out.println(maltese.getInfo());
    }
}

Kết quả vẫn là:

Breed is: Maltese Size is: Small Age is: 2 Color is: White

5. Kết luận

Hiểu rõ sự khác biệt giữa lớp (class) và đối tượng (object) giúp bạn nắm chắc nguyên lý cốt lõi của lập trình hướng đối tượng (OOP).

  • Lớp là bản thiết kế.
  • Đối tượng là biểu diễn cụ thể.
    Biết cách khai báo lớp, gán thuộc tính và gọi phương thức sẽ giúp bạn xây dựng phần mềm linh hoạt, dễ mở rộng và tái sử dụng cao.
