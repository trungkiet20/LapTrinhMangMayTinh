JavaScript: Hàm, Closure và Scope

October 17, 2025

Trong JavaScript, hàm (function) là khối mã có thể được gọi lại nhiều lần. Khi hiểu rõ scope (phạm vi biến) và closure, bạn sẽ viết được mã hiệu quả, tránh lỗi biến không mong muốn.

1️⃣ Hàm trong JavaScript

Bạn có thể khai báo hàm theo 2 cách:

// Cách 1: Khai báo thông thường
function sayHello(name) {
  console.log("Xin chào " + name);
}

// Cách 2: Biểu thức hàm
const greet = function(name) {
  console.log(`Chào bạn, ${name}`);
};

2️⃣ Scope – Phạm vi biến

Biến trong JavaScript có 3 loại scope chính:

  • Global Scope – Biến khai báo ngoài mọi hàm.
  • Function Scope – Biến chỉ tồn tại bên trong hàm.
  • Block Scope – (với let và const), tồn tại trong cặp ngoặc {}.
let x = 10; // global

function test() {
  let y = 20; // function scope
  if (true) {
    let z = 30; // block scope
    console.log(x, y, z); // 10, 20, 30
  }
  // console.log(z); // Lỗi! z không tồn tại ở đây
}
test();

3️⃣ Hoisting – Kéo lên đầu

JavaScript tự động “kéo” khai báo biến và hàm lên đầu phạm vi, nhưng không kéo giá trị gán.

console.log(a); // undefined
var a = 5;

// Với let/const thì sẽ lỗi ReferenceError

4️⃣ Closure – Hàm ghi nhớ môi trường của nó

Closure xảy ra khi một hàm con vẫn có thể truy cập biến của hàm cha sau khi hàm cha đã kết thúc.

function createCounter() {
  let count = 0;
  return function() {
    count++;
    console.log("Giá trị hiện tại:", count);
  };
}

const counter = createCounter();
counter(); // 1
counter(); // 2
counter(); // 3

🧠 Mỗi lần gọi createCounter(), bạn tạo ra một “môi trường riêng” mà hàm con vẫn ghi nhớ biến count.

5️⃣ Ứng dụng thực tế của closure

  • Tạo hàm đếm, lưu trạng thái mà không dùng biến toàn cục
  • Bảo vệ dữ liệu (encapsulation)
  • Tạo module nhỏ, tránh xung đột biến
