Java: Kế thừa và đa hình

October 17, 2025

Trong lập trình hướng đối tượng (OOP), kế thừa (inheritance) và đa hình (polymorphism) là hai khái niệm cực kỳ quan trọng giúp code linh hoạt, dễ mở rộng và tái sử dụng.

1. Kế thừa (Inheritance) là gì?

Kế thừa cho phép một lớp (class) con thừa hưởng các thuộc tính và phương thức của lớp cha. Điều này giúp tránh lặp code và dễ mở rộng tính năng.

class Animal {
    void sound() {
        System.out.println("Động vật phát ra âm thanh");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Gâu gâu!");
    }
}

public class Main {
    public static void main(String[] args) {
        Dog d = new Dog();
        d.sound(); // từ lớp cha
        d.bark();  // từ lớp con
    }
}

👉 Kết quả:

Động vật phát ra âm thanh
Gâu gâu!

Từ khóa extends

Dùng để khai báo rằng một lớp kế thừa từ lớp khác. Java chỉ hỗ trợ kế thừa đơn (một lớp chỉ có một cha).

2. Lớp trừu tượng (Abstract Class)

Khi bạn muốn định nghĩa cấu trúc chung nhưng không muốn cho phép tạo đối tượng trực tiếp, hãy dùng abstract.

abstract class Shape {
    abstract void draw(); // phương thức trừu tượng
}

class Circle extends Shape {
    void draw() {
        System.out.println("Vẽ hình tròn");
    }
}

public class Test {
    public static void main(String[] args) {
        Shape s = new Circle();
        s.draw();
    }
}

👉 abstract class có thể chứa cả phương thức trừu tượng (chưa có thân hàm) và phương thức bình thường.

3. Interface trong Java

Interface định nghĩa các hành vi (method) mà lớp khác phải triển khai. Một lớp có thể implements nhiều interface.

interface Flyable {
    void fly();
}

interface Swimmable {
    void swim();
}

class Duck implements Flyable, Swimmable {
    public void fly() {
        System.out.println("Con vịt bay được!");
    }
    public void swim() {
        System.out.println("Con vịt bơi dưới nước!");
    }
}

public class Demo {
    public static void main(String[] args) {
        Duck d = new Duck();
        d.fly();
        d.swim();
    }
}

Interface giúp tách biệt hành vi khỏi cấu trúc, hỗ trợ đa kế thừa hành vi.

4. Đa hình (Polymorphism)

Đa hình cho phép một đối tượng có thể mang nhiều hình dạng khác nhau — tùy thuộc vào ngữ cảnh.

Ví dụ: Một biến kiểu Animal có thể trỏ đến đối tượng Dog hoặc Cat.

class Animal {
    void makeSound() {
        System.out.println("Âm thanh chung của động vật");
    }
}

class Cat extends Animal {
    void makeSound() {
        System.out.println("Meo meo");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println("Gâu gâu");
    }
}

public class TestPoly {
    public static void main(String[] args) {
        Animal a1 = new Dog();
        Animal a2 = new Cat();

        a1.makeSound(); // Gâu gâu
        a2.makeSound(); // Meo meo
    }
}

👉 Đây là ví dụ của đa hình thời gian chạy (runtime polymorphism).

5. Lợi ích của kế thừa và đa hình

  • ✅ Tái sử dụng mã (code reuse).
  • ✅ Dễ mở rộng và bảo trì.
  • ✅ Dễ hiểu, gọn gàng, tránh lặp lại.
  • ✅ Cho phép xử lý nhiều loại đối tượng qua cùng giao diện.

6. Tổng kết

  • extends: kế thừa từ một lớp cha.
  • implements: triển khai từ một hoặc nhiều interface.
  • abstract class: lớp khung, không thể tạo trực tiếp.
  • Polymorphism: xử lý linh hoạt nhiều đối tượng khác nhau qua cùng kiểu tham chiếu.
