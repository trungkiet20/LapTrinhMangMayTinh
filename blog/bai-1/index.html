Bài 1 - Giới thiệu lập trình mạng và mô hình OSI

Đăng ngày 20/10/2025 lúc 00:00

1. Lập trình mạng là gì

Lập trình mạng là lĩnh vực giúp máy tính có thể giao tiếp với nhau thông qua kết nối mạng.
Các chương trình mạng thường có hai thành phần chính:

  • Client: là bên gửi yêu cầu (ví dụ: trình duyệt web, ứng dụng chat).
  • Server: là bên nhận yêu cầu và gửi phản hồi (ví dụ: máy chủ web, máy chủ game).

Lập trình mạng giúp ta:

  • Truyền dữ liệu giữa các thiết bị thông qua mạng LAN, WAN hoặc Internet.
  • Tạo ứng dụng client - server như: chat, chia sẻ file, game online, API.
  • Tìm hiểu cách dữ liệu được gửi đi, xử lý và phản hồi qua các tầng của mạng.

2. Mô hình OSI (Open Systems Interconnection)

Mô hình OSI là tiêu chuẩn do ISO đề xuất để mô tả cách dữ liệu di chuyển trong hệ thống mạng.
Mô hình gồm 7 tầng, từ tầng vật lý đến tầng ứng dụng.

Tầng Tên tầng Chức năng chính
7 Ứng dụng (Application) Cung cấp giao diện cho người dùng và ứng dụng
6 Trình bày (Presentation) Chuyển đổi định dạng dữ liệu, mã hóa, nén
5 Phiên (Session) Quản lý phiên giao tiếp giữa các tiến trình
4 Vận chuyển (Transport) Truyền dữ liệu tin cậy (TCP) hoặc nhanh (UDP)
3 Mạng (Network) Định tuyến, cấp phát địa chỉ IP
2 Liên kết dữ liệu (Data Link) Kiểm soát lỗi, khung dữ liệu
1 Vật lý (Physical) Truyền tín hiệu điện hoặc sóng radio

Thực tế, chúng ta thường sử dụng mô hình TCP/IP rút gọn còn 4 tầng:

  • Application
  • Transport
  • Internet
  • Network Access

Mô hình TCP/IP được dùng trong hầu hết các hệ thống mạng hiện nay, kể cả Internet.

3. Giao thức truyền thông (Protocol)

Giao thức là tập hợp các quy tắc quy định cách các thiết bị truyền và nhận dữ liệu.
Một số giao thức thông dụng:

Giao thức Mục đích sử dụng
HTTP / HTTPS Truyền dữ liệu web
FTP Truyền file
SMTP / POP3 / IMAP Gửi và nhận email
DNS Tra cứu tên miền
TCP / UDP Giao tiếp ở tầng vận chuyển
IP Giao tiếp ở tầng mạng

4. Socket là gì

Socket là một điểm cuối (endpoint) của kết nối giữa hai tiến trình qua mạng.
Mỗi socket có địa chỉ IP và số cổng (port).
Ví dụ: 127.0.0.1:8080

Có hai loại socket chính:

  • TCP Socket: kết nối có xác nhận, đảm bảo dữ liệu đến nơi.
  • UDP Socket: kết nối không xác nhận, nhanh nhưng không đảm bảo dữ liệu.

5. Ví dụ minh họa lập trình mạng với TCP Socket

Dưới đây là ví dụ cơ bản bằng Java về mô hình client - server:

File Server.java

import java.io.*;
import java.net.*;

public class Server {
    public static void main(String[] args) throws IOException {
        ServerSocket serverSocket = new ServerSocket(8080);
        System.out.println("Server đang lắng nghe tại cổng 8080...");

        Socket socket = serverSocket.accept();
        System.out.println("Client đã kết nối!");

        BufferedReader input = new BufferedReader(new InputStreamReader(socket.getInputStream()));
        PrintWriter output = new PrintWriter(socket.getOutputStream(), true);

        String message = input.readLine();
        System.out.println("Tin nhắn từ client: " + message);

        output.println("Server đã nhận: " + message);

        socket.close();
        serverSocket.close();
    }
}

File Client.java

import java.io.*;
import java.net.*;

public class Client {
    public static void main(String[] args) throws IOException {
        Socket socket = new Socket("localhost", 8080);
        System.out.println("Đã kết nối tới server!");

        BufferedReader input = new BufferedReader(new InputStreamReader(System.in));
        PrintWriter output = new PrintWriter(socket.getOutputStream(), true);
        BufferedReader serverInput = new BufferedReader(new InputStreamReader(socket.getInputStream()));

        System.out.print("Nhập tin nhắn gửi server: ");
        String message = input.readLine();
        output.println(message);

        String response = serverInput.readLine();
        System.out.println("Phản hồi từ server: " + response);

        socket.close();
    }
}

Khi chạy chương trình:

  1. Chạy Server.java trước.
  2. Sau đó chạy Client.java.
  3. Nhập tin nhắn và xem kết quả trên màn hình.

6. Kiến thức bổ sung

Kết nối TCP hoạt động như thế nào

  1. Three-way Handshake

    • Client gửi yêu cầu SYN đến server.
    • Server phản hồi SYN-ACK.
    • Client gửi ACK xác nhận. => Kết nối được thiết lập.

  2. Truyền dữ liệu

    • Dữ liệu được chia thành các gói (packet).
    • Mỗi gói có số thứ tự, đảm bảo gửi đúng thứ tự.

  3. Đóng kết nối

    • Cả hai bên gửi FIN và ACK để xác nhận ngắt kết nối.

7. Phân biệt TCP và UDP

Đặc điểm TCP UDP
Kiểu kết nối Có kết nối (connection-oriented) Không kết nối (connectionless)
Độ tin cậy Đảm bảo Không đảm bảo
Tốc độ Chậm hơn Nhanh hơn
Ứng dụng Web, Email, FTP Game, Video, Voice Chat

8. Ứng dụng thực tế của lập trình mạng

  • Ứng dụng chat thời gian thực
  • Ứng dụng truyền file
  • Dịch vụ API client-server
  • Game online nhiều người chơi
  • Giám sát hệ thống mạng
