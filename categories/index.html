Java: Đa hình (Polymorphism)

🧩 1. Khái niệm

  • Polymorphism có nghĩa là nhiều hình dạng.
  • Trong Java, cùng một phương thức có thể hoạt động khác nhau tùy theo đối tượng thực thi.
  • Có hai loại đa hình chính:
    1. Đa hình lúc biên dịch (Compile-time) → qua method overloading
    2. Đa hình lúc chạy (Runtime) → qua method overriding

⚙️ 2. Ví dụ về Đa hình lúc chạy (Runtime Polymorphism)

🔹 Ví dụ

// Lớp cha
class Animal {
    public void makeSound() {
        System.out.println("Động vật phát ra âm thanh...");
    }
}

// Lớp con 1
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Gâu gâu!");
    }
}

// Lớp con 2
class Cat extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Meo meo!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a1 = new Dog();  // tham chiếu kiểu Animal, đối tượng Dog
        Animal a2 = new Cat();  // tham chiếu kiểu Animal, đối tượng Cat

        a1.makeSound(); // gọi phiên bản của Dog
        a2.makeSound(); // gọi phiên bản của Cat
    }
}

Java: Tính đóng gói (Encapsulation)

Tính đóng gói (Encapsulation) là một trong bốn đặc tính cơ bản của lập trình hướng đối tượng (OOP) trong Java, bên cạnh kế thừa, đa hình và trừu tượng hóa.
Nó giúp bảo vệ dữ liệu và kiểm soát quyền truy cập vào các thuộc tính của lớp.

🧩 1. Khái niệm

  • Encapsulation nghĩa là gói gọn dữ liệu và các hành vi liên quan vào cùng một lớp.
  • Các thuộc tính (biến) trong lớp thường được ẩn đi (private), và chỉ có thể truy cập thông qua phương thức công khai (getter/setter).
  • Giúp đảm bảo dữ liệu nội bộ không bị thay đổi một cách trực tiếp, tránh sai sót và tăng tính bảo mật.

⚙️ 2. Cấu trúc cơ bản

🔹 Ví dụ

class Student {
    private String name;
    private int age;

    // Constructor
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter
    public String getName() {
        return name;
    }

    // Setter
    public void setName(String name) {
        this.name = name;
    }

    // Getter
    public int getAge() {
        return age;
    }

    // Setter có kiểm tra dữ liệu
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        } else {
            System.out.println("Tuổi không hợp lệ!");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student("Kiet", 20);
        System.out.println("Tên: " + s.getName());
        System.out.println("Tuổi: " + s.getAge());

        s.setAge(-5);  // dữ liệu không hợp lệ
        s.setAge(22);  // cập nhật hợp lệ
        System.out.println("Tuổi mới: " + s.getAge());
    }
}

Java: Ghi đè phương thức (Method Overriding) và Nạp chồng phương thức (Overloading)

Ghi đè (Overriding) và Nạp chồng (Overloading) là hai khái niệm cốt lõi trong lập trình hướng đối tượng (OOP) của Java, giúp tăng tính linh hoạt và tái sử dụng mã nguồn trong chương trình.

🧩 1. Nạp chồng phương thức (Method Overloading)

🔹 Khái niệm

  • Là việc định nghĩa nhiều phương thức cùng tên trong một lớp, nhưng khác nhau về số lượng hoặc kiểu dữ liệu của tham số.
  • Quyết định phương thức nào được gọi được xác định tại thời điểm biên dịch → compile-time polymorphism.

🔹 Cú pháp ví dụ

class Calculator {
    int add(int a, int b) {
        return a + b;
    }

    double add(double a, double b) {
        return a + b;
    }

    int add(int a, int b, int c) {
        return a + b + c;
    }
}

public class OverloadingExample {
    public static void main(String[] args) {
        Calculator c = new Calculator();
        System.out.println(c.add(3, 4));        // gọi phương thức int add(int,int)
        System.out.println(c.add(2.5, 3.7));    // gọi phương thức double add(double,double)
        System.out.println(c.add(1, 2, 3));     // gọi phương thức int add(int,int,int)
    }
}

Java: Lớp trừu tượng (Abstract Class) và Giao diện (Interface)

Lớp trừu tượng (Abstract Class) và Giao diện (Interface) là hai khái niệm quan trọng trong lập trình hướng đối tượng (OOP) của Java, giúp định nghĩa hành vi chung cho các lớp con mà không cần triển khai chi tiết ngay từ đầu.

🧩 1. Lớp trừu tượng (Abstract Class)

🔹 Khái niệm

  • Là lớp không thể tạo đối tượng trực tiếp (không thể new).
  • Dùng để định nghĩa khung cho các lớp con.
  • Có thể chứa:
    • Phương thức trừu tượng (abstract) → không có thân hàm, bắt buộc lớp con phải override.
    • Phương thức bình thường → có thân hàm.
    • Thuộc tính, constructor, hằng số,…

🔹 Cú pháp

abstract class Animal {
    String name;

    Animal(String name) {
        this.name = name;
    }

    abstract void makeSound(); // phương thức trừu tượng

    void eat() {
        System.out.println(name + " is eating.");
    }
}

🔹 Kế thừa và triển khai

class Dog extends Animal {
    Dog(String name) {
        super(name);
    }

    @Override
    void makeSound() {
        System.out.println("Woof woof!");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal a = new Dog("Buddy");
        a.eat();
        a.makeSound();
    }
}

Java: Interface (Giao diện)

Interface (Giao diện) là một khái niệm quan trọng trong lập trình hướng đối tượng (OOP) của Java, thường được sử dụng để định nghĩa hành vi chung cho nhiều lớp mà không cần quan tâm cách chúng được triển khai.
Nó giúp tăng tính linh hoạt, khả năng mở rộng và tái sử dụng mã trong các chương trình lớn.

🧩 1. Khái niệm

  • Interface là một bản mô tả (contract) các hành vi mà lớp phải thực hiện.
  • Một interface chỉ chứa phương thức trừu tượng (abstract methods) và hằng số (constants).
  • Các lớp implements interface phải định nghĩa lại (override) toàn bộ các phương thức trong đó.

⚙️ 2. Cấu trúc cơ bản

🔹 Ví dụ

// Định nghĩa interface
interface Animal {
    void makeSound();  // phương thức trừu tượng
    void eat();
}

// Lớp Dog triển khai interface
class Dog implements Animal {
    public void makeSound() {
        System.out.println("Gâu gâu!");
    }

    public void eat() {
        System.out.println("Chó đang ăn xương.");
    }
}

// Lớp Cat triển khai interface
class Cat implements Animal {
    public void makeSound() {
        System.out.println("Meo meo!");
    }

    public void eat() {
        System.out.println("Mèo đang ăn cá.");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal dog = new Dog();
        Animal cat = new Cat();

        dog.makeSound();
        dog.eat();

        cat.makeSound();
        cat.eat();
    }
}
