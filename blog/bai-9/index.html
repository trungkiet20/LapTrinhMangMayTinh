Java: Đa hình (Polymorphism)

Đăng ngày 20/10/2025 lúc 10:45

Đa hình (Polymorphism) là một trong bốn đặc tính cơ bản của lập trình hướng đối tượng (OOP) trong Java, cho phép một hành động có thể thể hiện theo nhiều cách khác nhau.
Nhờ đa hình, chương trình trở nên linh hoạt, dễ mở rộng và tái sử dụng tốt hơn.

🧩 1. Khái niệm

  • Polymorphism có nghĩa là nhiều hình dạng.
  • Trong Java, cùng một phương thức có thể hoạt động khác nhau tùy theo đối tượng thực thi.
  • Có hai loại đa hình chính:
    1. Đa hình lúc biên dịch (Compile-time) → qua method overloading
    2. Đa hình lúc chạy (Runtime) → qua method overriding

⚙️ 2. Ví dụ về Đa hình lúc chạy (Runtime Polymorphism)

// Lớp cha
class Animal {
    public void makeSound() {
        System.out.println("Động vật phát ra âm thanh...");
    }
}

// Lớp con 1
class Dog extends Animal {
    @Override
    public void makeSound() {
        System.out.println("Gâu gâu!");
    }
}

👈 (Mã nguồn đã được rút gọn để minh họa)

🔄 3. Ví dụ về Đa hình lúc biên dịch

Đa hình lúc biên dịch đạt được thông qua Method Overloading (Nạp chồng phương thức).

🔍 4. Lợi ích của Polymorphism

  • ✅ Linh hoạt: Một hành động có nhiều cách thể hiện.
  • ✅ Mở rộng: Thêm lớp mới dễ dàng.
  • ✅ Bảo trì: Giảm sự phụ thuộc.
