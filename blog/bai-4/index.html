Java: Xử lý ngoại lệ và logging

Đăng ngày 17/10/2025 lúc 09:30

Đóng gói (Encapsulation) là một trong bốn trụ cột chính của lập trình hướng đối tượng (OOP), cùng với Kế thừa, Đa hình, và Trừu tượng.
Nó giúp bảo vệ dữ liệu bên trong đối tượng, chỉ cho phép truy cập thông qua phương thức an toàn thay vì trực tiếp thao tác với biến.

🧩 1. Khái niệm Encapsulation

Encapsulation là cơ chế che giấu thông tin (data hiding), trong đó:

  • Các thuộc tính (fields) của lớp được đặt ở chế độ private.
  • Việc truy cập và thay đổi dữ liệu sẽ được thực hiện qua getter và setter (phương thức công khai).

Điều này đảm bảo rằng dữ liệu chỉ được thay đổi một cách có kiểm soát.

🔹 Ví dụ minh họa cơ bản

class Person {
    private String name;
    private int age;

    // Getter - lấy giá trị
    public String getName() {
        return name;
    }

    // Setter - gán giá trị
    public void setName(String newName) {
        name = newName;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int newAge) {
        if (newAge >= 0) {
            age = newAge;
        } else {
            System.out.println("Tuổi không hợp lệ!");
        }
    }
}

public class EncapsulationDemo {
    public static void main(String[] args) {
        Person p = new Person();
        p.setName("Kiệt");
        p.setAge(25);

        System.out.println("Tên: " + p.getName());
        System.out.println("Tuổi: " + p.getAge());
    }
}

📤 Kết quả khi chạy:

Tên: Kiệt
Tuổi: 25

⚙️ 2. Tại sao nên dùng Encapsulation?

Ưu điểm Giải thích
🔒 Bảo mật dữ liệu Không cho phép truy cập trực tiếp vào các biến nội bộ.
🧱 Kiểm soát hợp lệ dữ liệu Có thể kiểm tra dữ liệu đầu vào thông qua setter.
🔄 Dễ bảo trì và mở rộng Thay đổi cách lưu trữ dữ liệu mà không ảnh hưởng đến bên ngoài.
🧩 Tái sử dụng mã tốt hơn Dễ quản lý, tái sử dụng trong các dự án lớn.

🔮 3. Quy tắc khi áp dụng Encapsulation

  1. Khai báo các biến thành viên là private.
  2. Cung cấp phương thức public getter và setter để truy cập dữ liệu.
  3. Có thể áp dụng kiểm tra hợp lệ (validation) trong setter để đảm bảo tính toàn vẹn dữ liệu.

🧠 4. Ví dụ nâng cao với kiểm tra dữ liệu

class BankAccount {
    private double balance;

    // Nạp tiền
    public void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        } else {
            System.out.println("Số tiền nạp không hợp lệ!");
        }
    }

    // Rút tiền
    public void withdraw(double amount) {
        if (amount > 0 && amount <= balance) {
            balance -= amount;
        } else {
            System.out.println("Rút tiền thất bại!");
        }
    }

    // Xem số dư
    public double getBalance() {
        return balance;
    }
}

public class BankDemo {
    public static void main(String[] args) {
        BankAccount account = new BankAccount();

        account.deposit(1000);
        account.withdraw(300);
        account.withdraw(800); // Thử vượt quá số dư

        System.out.println("Số dư hiện tại: " + account.getBalance());
    }
}

📤 Kết quả:

Rút tiền thất bại!
Số dư hiện tại: 700.0

💡 5. So sánh: Public vs Private vs Protected

Phạm vi Mô tả Truy cập được ở đâu
public Truy cập mọi nơi Trong & ngoài gói (package)
private Chỉ trong cùng một lớp Không thể truy cập từ lớp khác
protected Cho phép truy cập từ lớp con Cùng package hoặc lớp kế thừa

🚀 6. Kết luận

  • ✅ Encapsulation giúp bảo vệ dữ liệu tránh bị thay đổi ngoài ý muốn.
  • 🧱 Dữ liệu chỉ nên truy cập thông qua getter và setter.
  • 💡 Áp dụng nguyên lý này giúp mã Java của bạn an toàn, rõ ràng, và chuyên nghiệp hơn.
