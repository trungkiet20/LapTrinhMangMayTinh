Java: Tính đóng gói (Encapsulation)

Đăng ngày 20/10/2025 lúc 09:45

Tính đóng gói (Encapsulation) là một trong bốn đặc tính cơ bản của lập trình hướng đối tượng (OOP) trong Java, bên cạnh kế thừa, đa hình và trừu tượng hóa.
Nó giúp bảo vệ dữ liệu và kiểm soát quyền truy cập vào các thuộc tính của lớp.

🧩 1. Khái niệm

  • Encapsulation nghĩa là gói gọn dữ liệu và các hành vi liên quan vào cùng một lớp.
  • Các thuộc tính (biến) trong lớp thường được ẩn đi (private), và chỉ có thể truy cập thông qua phương thức công khai (getter/setter).
  • Giúp đảm bảo dữ liệu nội bộ không bị thay đổi một cách trực tiếp, tránh sai sót và tăng tính bảo mật.

⚙️ 2. Cấu trúc cơ bản

🔹 Ví dụ

class Student {
    private String name;
    private int age;

    // Constructor
    public Student(String name, int age) {
        this.name = name;
        this.age = age;
    }

    // Getter
    public String getName() {
        return name;
    }

    // Setter
    public void setName(String name) {
        this.name = name;
    }

    // Getter
    public int getAge() {
        return age;
    }

    // Setter có kiểm tra dữ liệu
    public void setAge(int age) {
        if (age > 0) {
            this.age = age;
        } else {
            System.out.println("Tuổi không hợp lệ!");
        }
    }
}

public class Main {
    public static void main(String[] args) {
        Student s = new Student("Kiet", 20);
        System.out.println("Tên: " + s.getName());
        System.out.println("Tuổi: " + s.getAge());

        s.setAge(-5);  // dữ liệu không hợp lệ
        s.setAge(22);  // cập nhật hợp lệ
        System.out.println("Tuổi mới: " + s.getAge());
    }
}

Kết quả:

Tên: Kiet
Tuổi: 20
Tuổi không hợp lệ!
Tuổi mới: 22

🔍 3. Lợi ích của Encapsulation

Lợi ích Mô tả
🔒 Bảo mật dữ liệu Giữ dữ liệu ở trạng thái riêng tư, tránh bị thay đổi ngoài ý muốn
⚙️ Dễ bảo trì Có thể thay đổi cách lưu trữ dữ liệu mà không ảnh hưởng lớp khác
🧱 Kiểm soát truy cập Cho phép thêm logic khi thay đổi dữ liệu (qua setter)
🧠 Dễ mở rộng Có thể thêm kiểm tra dữ liệu hoặc xử lý mà không phá vỡ mã hiện có

🧠 4. Thực tế trong Java

Các class trong thư viện Java sử dụng Encapsulation rất nhiều, ví dụ:

String s = "Hello";
System.out.println(s.length()); // truy cập qua phương thức, không chạm trực tiếp biến nội bộ

Bạn không thể truy cập trực tiếp biến value trong String vì nó là private.
Chỉ có thể lấy độ dài, ký tự,… thông qua các phương thức public.

✅ 5. Tổng kết

Đặc điểm Ý nghĩa
Private field Ẩn dữ liệu nội bộ
Getter/Setter Cung cấp cách truy cập an toàn
Kiểm soát logic Cho phép xác thực dữ liệu trước khi thay đổi
Tăng bảo mật & tính toàn vẹn dữ liệu Giúp chương trình ổn định và dễ bảo trì

💡 Encapsulation chính là bước đầu tiên để viết code Java chuẩn hướng đối tượng, an toàn và dễ mở rộng.
